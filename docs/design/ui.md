# 画面設計（UI）

MVP は2画面 + 1モーダル：チャット（既定 `/`）、登録（`/register`）、原文閲覧モーダル。

---

## 1. 共通レイアウト（`app/layout.tsx`）

```
┌───────────────────────────────────────────────┐
│  メルマガ相談チャット      [ チャット ] [ 登録 ]   │  ← ヘッダ + ナビ
├───────────────────────────────────────────────┤
│                 （各ページ）                    │
└───────────────────────────────────────────────┘
```

- ナビは2リンク（現在地をハイライト）。スタイルは最小限（Tailwind 推奨、素の CSS も可）。

---

## 2. チャット画面（`app/page.tsx`）

### ワイヤーフレーム

```
┌───────────────────────────────────────────────┐
│ 参照期間: [ 2024-01-01 ] 〜 [ 2024-12-31 ] (任意) │ ← DateRangeFilter（折りたたみ可）
├───────────────────────────────────────────────┤
│  🧑 新規事業の価格設定はどう考えるべき？           │
│                                               │
│  🤖 まず…（ストリーミングで逐次表示）…            │
│     ───────────────────────────               │
│     出典:                                      │ ← SourceList
│      • 週刊〇〇通信 第12号 (2024-01-15) 山田太郎  │
│        [ 原文を見る ]                          │ ← クリックで RawNoteModal
│      • 〇〇レター (2024-03-02) 佐藤花子          │
│        [ 原文を見る ]                          │
├───────────────────────────────────────────────┤
│ [ 質問を入力…                        ] [ 送信 ] │ ← MessageInput
└───────────────────────────────────────────────┘
```

### コンポーネント構成（`components/chat/`）

| コンポーネント | 役割 | 主な props/state |
|---|---|---|
| `ChatContainer` | 状態管理（`messages`, `dateFrom/To`, `isStreaming`, `rawModal`） | クライアント state |
| `DateRangeFilter` | 参照期間 from/to（任意） | `value`, `onChange` |
| `MessageList` | 会話履歴の描画 | `messages` |
| `MessageItem` | 1メッセージ。assistant は本文 + `SourceList` | `message` |
| `SourceList` | 出典（件名・発行日・著者）＋「原文を見る」 | `sources`, `onOpenRaw` |
| `RawNoteModal` | 原文ノート表示（`/api/newsletters/{id}/raw` を取得） | `newsletterId`, `onClose` |
| `MessageInput` | 入力 + 送信（Enter 送信 / Shift+Enter 改行） | `onSend`, `disabled` |

### 送受信フロー

1. 送信時、`messages` に user を追加し、空の assistant を追加。
2. `fetch("/api/chat", {method:"POST", body:{messages, dateFrom, dateTo}})`。
3. `response.body` を `getReader()` で読み、`␞__SOURCES__␞` までを本文として逐次反映。
4. 区切り後の JSON を `sources` として該当 assistant に格納・描画（各 source は `newsletterId` を保持）。
5. 「原文を見る」→ `RawNoteModal` が `GET /api/newsletters/{newsletterId}/raw` を取得して表示。
6. `␞__ERROR__␞` 検出時はエラー表示。

### 状態

| 状態 | UI |
|---|---|
| 初期 | 空の履歴 + 入力欄。使い方の一文（任意） |
| ストリーミング中 | 入力欄 disable、assistant に「…」 |
| 完了 | 出典リスト + 原文リンク表示、入力欄 enable |
| 原文モーダル | オーバーレイで原文（メタ + 本文）。閉じるで戻る |
| エラー | 該当メッセージに赤字 + 再試行可 |

## 3. 登録画面（`app/register/page.tsx`）

### ワイヤーフレーム

```
┌───────────────────────────────────────────────┐
│  メルマガを登録                                  │
│  件名*       [______________________________]  │
│  発行日*     [ 2024-01-15 ▾ ]                   │
│  号数        [____________]  著者 [___________] │
│  カテゴリ     [____________]                    │
│  タグ        [ 価格設定 ×][ 新規事業 ×][＋追加]   │
│  発行元      [______________________________]  │
│  本文*                                         │
│  ┌───────────────────────────────────────┐    │
│  │ （メルマガ本文を貼り付け）                  │    │
│  └───────────────────────────────────────┘    │
│                                  [ 登録する ]   │
│  ⏳ 整理ノートを生成して登録中…                  │ ← 生成待ち表示
│  ✅ 「週刊〇〇通信 第12号」を登録しました          │
└───────────────────────────────────────────────┘
```

### コンポーネント（`components/register/`）

| コンポーネント | 役割 |
|---|---|
| `NewsletterForm` | フォーム状態（title, issueDate, issueNo, author, category, tags[], source, body）、送信、結果表示 |
| `TagInput` | タグの追加/削除（任意・簡易） |

### 送信フロー

1. クライアントで必須チェック（件名・発行日・本文）。
2. `fetch("/api/ingest", {method:"POST", body:{...}})`。
3. **処理は数十秒かかりうる**（整理ノート生成 + VS インデックス）。送信ボタンを「整理ノートを生成して登録中…」にし二重送信防止。
4. 成功（200）→ 成功メッセージ + フォームクリア（連続登録しやすく）。
5. 409（重複）→「同じ本文が登録済み」。
6. 400/500 → 理由表示（入力は保持）。

### 状態

| 状態 | UI |
|---|---|
| 入力中 | 必須未入力なら送信 disable |
| 送信中 | 「生成して登録中…」、二重送信防止 |
| 成功 | 成功メッセージ + クリア |
| エラー | 理由表示、入力保持 |

## 4. 原文閲覧モーダル（RawNoteModal）

```
┌──────────────── 原文ノート ────────────────┐
│ 週刊〇〇通信 第12号                      [×] │
│ 発行日: 2024-01-15 ｜ 著者: 山田太郎        │
│ カテゴリ: マーケティング ｜ タグ: 価格設定…   │
│ ───────────────────────────────────────── │
│ （原文本文をそのまま表示・スクロール）          │
│                                           │
└───────────────────────────────────────────┘
```

- 出典の「原文を見る」から開く。`GET /api/newsletters/{id}/raw` のメタ + 本文を表示。
- 発行時点の文脈・事実確認の用途。読み取り専用。

---

## 5. アクセシビリティ / 操作性（最小限）

- `label` と入力の関連付け、必須は `aria-required`。
- チャット送信は Enter、改行は Shift+Enter。
- 送信中は明確に disable（特に ingest は時間がかかるため進行表示）。
- モーダルは Esc で閉じる、フォーカストラップ（簡易）。
- 色のみに依存しないエラー表示（アイコン/文言併用）。

---

## 6. スタイル方針

- MVP は装飾を最小化。可読性・余白優先。
- Tailwind 採用時もユーティリティは最小限。コンポーネント分割を優先し、後からスタイル強化できる構造に。
